Claude Code Session Tracker

A tool to monitor all running Claude Code sessions from one place. See status, summaries, and jump to any session in tmux.

Requirements

  • See all sessions in one pane
  • Summary of each session
  • Session status (active / idle / waiting)
  • Jump to tmux pane
  • Single-user, local only

Option 1 — Hooks + SQLite + Terminal UI Recommended

Claude Code hooks fire on session events and write to a shared SQLite database. A TUI app (running in its own tmux pane) reads from SQLite and renders a live dashboard. Jump-to-session is native since you're already in tmux.

Pros

  • Lives in tmux — no context switch
  • Native jump-to-pane (instant)
  • No browser, no extra processes
  • SQLite handles concurrent writes safely
  • Textual makes polished TUIs quickly

Cons

  • Textual / Bubble Tea is a library to learn
  • Harder to do rich visuals (charts, etc.)
  • Resizing can be finicky in tmux

Option 2 — Hooks + SQLite + Local Web Dashboard

Same hook-based data collection into SQLite. A small Python HTTP server serves an HTML dashboard on localhost with auto-refreshing via polling. Tmux jumping works through a server-side API endpoint that calls tmux select-pane.

Option 3 — Lightweight: Hooks + tmux Status Pane

Hooks write session state to simple JSON files under ~/.claude/tracking/. A small shell or Python script runs in a dedicated tmux pane, polling those files and rendering a compact table. Minimal moving parts.

Pros

  • Simplest to build (could be ~100 lines total)
  • Zero dependencies beyond Python stdlib
  • Fits naturally in tmux workflow
  • Easy to understand and modify

Cons

  • Limited interactivity (no click, no expand)
  • Harder to show detailed summaries
  • JSON files can have race conditions under heavy writes
  • Basically a watch loop

Shared: Hook Data Collection

All three options use the same hooks layer. Here's what gets captured:

Hook Events

  • SessionStart — session created, capture tmux pane ID via $TMUX_PANE
  • UserPromptSubmit — user sent a prompt (includes prompt text for summary)
  • PostToolUse — tool finished (async; captures tool name + file/command detail)
  • Stop — Claude finished responding, now idle waiting for input
  • PermissionRequest — session blocked, waiting for user to approve a tool
  • SessionEnd — session terminated

Data Available Per Event

  • session_id — unique session identifier
  • cwd — working directory (i.e. project path)
  • hook_event_name — which event fired
  • transcript_path — path to full session transcript
  • tool_name — Edit, Bash, Read, Grep, etc. (tool events only)
  • tool_input — file path, command, pattern, etc. (tool events only)
  • prompt — user's prompt text (UserPromptSubmit only)
  • $TMUX_PANE — env var with the tmux pane ID (e.g. %5)

Session Statuses

  • ● Active — Claude is working (tool calls, responding)
  • ● Idle — waiting for user input
  • ● Waiting — blocked on permission approval
  • ● Ended — session terminated

Example Hook Config

// In ~/.claude/settings.json { "hooks": { "SessionStart": [{ "hooks": [{ "type": "command", "command": "python3 ~/track.py" }] }], "UserPromptSubmit": [{ "hooks": [{ "type": "command", "command": "python3 ~/track.py" }] }], "PostToolUse": [{ "hooks": [{ "type": "command", "command": "python3 ~/track.py", "async": true }] }], "Stop": [{ "hooks": [{ "type": "command", "command": "python3 ~/track.py" }] }], "PermissionRequest": [{ "hooks": [{ "type": "command", "command": "python3 ~/track.py" }] }], "SessionEnd": [{ "hooks": [{ "type": "command", "command": "python3 ~/track.py" }] }] } }

Tmux Jump

The hook captures $TMUX_PANE at session start. To jump to a session:

tmux select-window -t %5 # switch to the window containing pane %5 tmux select-pane -t %5 # focus that specific pane
